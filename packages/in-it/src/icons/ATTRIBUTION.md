# Icon Attribution

## Lucide Icons

The built-in icons in in-it are derived from [Lucide Icons](https://lucide.dev),
a community-driven fork of [Feather Icons](https://feathericons.com).

SVG path data has been copied into this project — there is no runtime dependency
on the Lucide package. This approach keeps in-it's dependency footprint minimal
while providing high-quality, consistent iconography.

## Future Plans

We currently bundle a curated subset of Lucide icons for the built-in components.
Long-term, we'd love to develop a custom icon set tailored to in-it's design
language. If you're a designer interested in contributing, please reach out via
[GitHub Discussions](https://github.com/kotsumocom/in-it/discussions) — we'd
love to hear from you.
