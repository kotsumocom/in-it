# **in-it: MVP Specification (Phase 1)**

## **1. MVP の目的 (Goal)**

**検証仮説:**
「生徒は、事前の『要点（Gist）』把握と、AI による『効率化支援』があれば、授業の退屈さと宿題の苦痛を軽減できるか？」

**制約事項:**

- **Zero-Management:** 人力による個別指導は行わない（創業者による手動対応も最小限にし、AI で完結させる）。
- **Mobile First:** ユーザー（小学生・中学生・高校生）は主にスマホからアクセスする。

## **2. ターゲットユーザー (Alpha)**

- 小学生・中学生・高校生（テストユーザー 10 名程度）
- 課題: 「宿題が終わらない」「授業が暇すぎる」

## **3. 機能要件 (Functional Requirements)**

### **3.1 認証・ユーザー管理**

- **必須:** メールアドレス/パスワード認証 (Supabase Auth)
- **簡易:** ニックネーム登録のみ。

### **3.2 "The Map" (学習ロードマップ機能)**

- **機能:** 小学校〜高校までの全単元を「スキルツリー（地図）」として可視化。
  - ユーザーは自分の学年に関係なく、先の単元（未来）へどんどん進める（Acceleration）。
  - もちろん、「明日やる単元」をピンポイントで予習することも可能。
- **UI:** Duolingo やゲームのような「マップ型 UI」。進捗が可視化され、クリアしていく達成感を演出。
- **コンテンツ内容:** まずは**「数学」**のみに集中的に投下する。
  - 数学は構造化しやすく、短い動画での解説（Gist）とも相性が良いため、教材制作のコスパが最も良い。

### **3.3 ダッシュボード (My Status)**

- **機能:** 「現在の到達レベル（Current Academic Level）」の可視化。
- **表示イメージ:** 「中 3・5 月レベル」のように、学校の標準カリキュラムと比較してどこまで進んでいるかを表示。
- **ロジック:** 完了した単元のうち、最も進んでいるものの「標準学習時期」を表示する。
  - これにより「学校より半年先に進んでいる」という優越感（Acceleration）を可視化する。

## **4. 技術スタック案 (Tech Stack)**

- **Runtime:** Deno 2.x
- **API Framework:** Hono (Standardized API for Web & Future Native App)
- **Frontend (Web MVP):** Fresh
- **Backend/DB:** Supabase (PostgreSQL, Auth)
- **Hosting:** Deno Deploy (Edge hosting)
- **(Future AI):** Google Gemini API (Phase 2 以降で導入)

## **5. 開発スコープ外 (Out of Scope for MVP)**

- **AI Gear (スマート処理アシスタント):**
  - 宿題の写真を撮って解説してくれる AI 機能は、Phase 2 以降で実装する。
  - MVP 段階では「解説コンテンツ（Gist）」の質だけで勝負する。
- **ランダム復習機能 (Flash-Tok):**
  - TikTok 風の UI で、過去に学んだ単元の「要点カード」がランダムに流れてくる機能。
  - スワイプでサクサク復習し、将来の Flutter アプリ版での実装を想定。
- **ネイティブアプリ化 (Flutter):**
  - PWA ではなく、日本市場に合わせてストア公開できるネイティブアプリを Flutter で開発する（Phase 3 以降）。
- コミュニティ機能（まずは個人の効率化に集中）
- 複雑な学習管理システム（LMS）
- 決済機能（Alpha 版は無料または手動入金で対応）

## **6. 成功指標 (Success Metrics)**

- **DAU (Daily Active Users):** コンテンツを毎日見に来るか？
- **Retention:** 1 週間継続して利用するか？
- **Time Savings:** ユーザー主観で「楽になった」というフィードバックが得られるか？
