# 認証・登録フロー UI仕様書

## 概要

メンター（講師）向けの認証フロー。
Supabase Auth使用、メール/パスワード認証。

---

## フロー全体図

```
[トップ/一覧] → [登録] → [メール確認] → [プロフィール入力] → [決済] → [公開]
                   ↓
              [ログイン] → [マイページ]
```

---

## 画面一覧

### 1. 登録画面（Sign Up）

**入力項目:**
| 項目 | 必須 | バリデーション |
|------|------|----------------|
| メールアドレス | ✓ | 形式チェック |
| パスワード | ✓ | 8文字以上 |
| パスワード確認 | ✓ | 一致確認 |

**UI要素:**
- ロゴ（イニット）
- フォーム（シングルカラム）
- 「登録する」ボタン（Primary）
- 「ログインはこちら」リンク
- 利用規約・プライバシーポリシーへのリンク

**エラー表示:**
- フィールド下にインラインエラー
- Material Designのエラースタイル（赤）

---

### 2. ログイン画面（Sign In）

**入力項目:**
| 項目 | 必須 |
|------|------|
| メールアドレス | ✓ |
| パスワード | ✓ |

**UI要素:**
- ロゴ
- フォーム
- 「ログイン」ボタン
- 「パスワードを忘れた方」リンク
- 「新規登録はこちら」リンク

---

### 3. メール確認待ち画面

**内容:**
- 「確認メールを送信しました」メッセージ
- 送信先メールアドレス表示
- 「メールを再送信」ボタン
- 「メールが届かない場合」ヘルプリンク

---

### 4. パスワードリセット

**Step 1: メール入力**
- メールアドレス入力
- 「リセットリンクを送信」ボタン

**Step 2: 新パスワード設定**
- 新パスワード入力
- 確認入力
- 「パスワードを変更」ボタン

---

## 登録後フロー

```
メール確認完了
    ↓
プロフィール入力（必須項目のみ）
    ↓
Stripe Checkout（月額1,000円）
    ↓
プロフィール公開
```

---

## 認証状態による表示分岐

| 状態 | ヘッダー表示 |
|------|--------------|
| 未ログイン | 「登録」「ログイン」ボタン |
| ログイン済み（未決済） | 「マイページ」「プランを選択」 |
| ログイン済み（有料会員） | 「マイページ」アイコン |

---

## Supabase Auth設定

- Provider: Email
- メール確認: 必須
- パスワード最低文字数: 8
- リダイレクトURL設定
