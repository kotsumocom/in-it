# 講師プロフィールページ UI仕様書

## 概要

講師の「簡易ホームページ」として機能する公開プロフィールページ。
toC向け、Materialデザインベースのプレミアムなデザイン。

---

## デザイン方針

| 項目 | 方針 |
|------|------|
| ベース | Material Design 3 |
| レイアウト | **シングルカラム**（2カラム禁止） |
| 角 | 角ばったエッジ（Firebase/GitLab風） |
| カラー | 寒色系（青〜ティール） |

---

## ブランディング

| 項目 | 内容 |
|------|------|
| サービス名 | **in-it** |
| 読み方 | イニット |
| ロゴ | カタカナ「イニット」またはシンボルロゴ |
| ターゲット | 日本語圏 |

---

## ページ構成（ワイヤー）

### 1. ヒーローセクション
- プロフィール画像（円形 or 角丸正方形）
- 名前 + 本人確認バッジ
- タグライン（短いキャッチコピー）

### 2. カテゴリタグ
- ピル型、**単一カラー**
- 例: `プログラミング` `Python` `副業`

### 3. CTAボタン
- 「予約する」「相談する」等
- 外部リンクへ遷移

### 4. 自己紹介
- Markdownサポート

### 5. 外部リンク
- SNS、予約ページ、Zoom等のアイコンボタン

### 6. 実績/ポートフォリオ
- カード型、シングルカラム
- 縦スクロール

### 7. レビュー（Phase 2）
- 星評価 + レビュー

---

## 本人確認機能

| レベル | バッジ |
|--------|--------|
| 未確認 | なし |
| メール確認済み | ✓（グレー） |
| 本人確認済み | ✓（青） |
| プロ認定 | ⭐（ゴールド） |

---

## レビュー機能（Phase 2）

### 開示請求対応
- 投稿者のIPアドレス・メールアドレスをログ保存
- 悪質投稿への開示請求時に対応可能な設計
- プライバシーポリシーに明記
